[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture to see how it works under the hood.
(pause)
First, every user query passes through our Security Checkpoint node, which scrubs sensitive PII like emails and phone numbers and blocks injection attacks.
If clean, the request goes to the Orchestrator, which is our central coordinator agent.
The Orchestrator delegates tasks using AgentTool calls to two specialized sub-agents: the diet planner, which structures daily meal recipes, and the grocery specialist, which compiles the shopping list.
These sub-agents connect to a local MCP Server to search recipe catalogs and query the current pantry items.
Before the final plan is returned, it hits our human-in-the-loop approval node. If the estimated cost exceeds fifty dollars, the workflow pauses to ask for your explicit approval before presenting the plan.

[3:00 — SECURITY & MCP]
Our security checkpoint uses regex matching and keyword detection, logging every decision in structured JSON format for audit purposes.
Additionally, our MCP server exposes tools like get_pantry_items and search_recipes, allowing the agent to seamlessly interact with local databases.
